## 0. Purpose

TP01 is the daily-use anchor extraction workflow for turning one or more reference images into a reusable Anchor Set.

TP01 produces three separated anchors:

```text
AnchorA.PNG = layout grammar only
AnchorB.PNG = style DNA only
AnchorC.PNG = asset / object vocabulary only
```

This file is the executable workflow. It defines source authority, execution order, stop points, runtime contracts, validation, manifest, package governance, failure handling, and failed-but-useful salvage rules.

Concrete generation wording lives in:

```text
TP01_PROMPTS.md
```

Failed-image classification lives in:

```text
TP01_FAILURE_SALVAGE.md
```

This split is intentional:

```text
Tp01.md = stable protocol / operating system
TP01_PROMPTS.md = editable extraction prompt module
TP01_FAILURE_SALVAGE.md = failed-image salvage and retry classification module
```

When prompt behavior needs refinement, update `TP01_PROMPTS.md` first. When failed outputs need role-classification rules, update `TP01_FAILURE_SALVAGE.md` first. Do not rewrite this core workflow unless the protocol itself changes.

## 4. Invocation Contract

When the user says `tp01` and does not attach a TP01 file:

1. Fetch repository metadata from `hoiyu915-droid/Tp01`.
2. Read `TP01_SOURCE_LOCK.md`.
3. Read `Tp01.md`, including SHA / metadata.
4. Read `TP01_PROMPTS.md`, including SHA / metadata.
5. Read `TP01_FAILURE_SALVAGE.md`, including SHA / metadata.
6. Read AnchorA / AnchorB / AnchorC from the canonical raw links.
7. Verify all source components are available:
   - `Tp01.md`
   - `TP01_PROMPTS.md`
   - `TP01_FAILURE_SALVAGE.md`
   - `AnchorA.PNG`
   - `AnchorB.PNG`
   - `AnchorC.PNG`
8. **STOP POINT 1 — Anchor Read Report**
   - Before any runtime or generation, output a report confirming whether AnchorA / B / C were actually read as visual content.
   - This report must distinguish:
     - file existence / metadata
     - SHA availability
     - actual visual-read success
     - unresolved failures
   - If any anchor image was not actually read visually, stop the TP01 run here.
9. Inspect the current user reference / task only after Stop Point 1 passes.
10. Generate current-run runtime files from the current source inputs:
    - `AnchorA.runtime.md`
    - `AnchorB.runtime.md`
    - `AnchorC.runtime.md`
11. **STOP POINT 1R — Runtime Report**
    - Evaluate whether each runtime was derived from the current source input.
    - Confirm whether:
      - AnchorA runtime is layout grammar only
      - AnchorB runtime is style DNA only
      - AnchorC runtime is asset / object vocabulary only
      - the three runtimes are sufficiently separated
      - no runtime silently reverted to canonical baseline content
    - Do not generate anchor images before this report passes.
12. Create `SOURCE_DIR / SOURCE_INDEX.md` draft if creating a package.
13. Generate `AnchorA.PNG` sequentially from `AnchorA.runtime.md` and `TP01_PROMPTS.md` AnchorA prompt rules.
14. Validate `AnchorA.PNG`.
15. If AnchorA fails, run `TP01_FAILURE_SALVAGE.md` failed-but-useful gate before retrying.
16. Generate `AnchorB.PNG` sequentially from `AnchorB.runtime.md` and `TP01_PROMPTS.md` AnchorB prompt rules.
17. Validate `AnchorB.PNG`.
18. If AnchorB fails, run `TP01_FAILURE_SALVAGE.md` failed-but-useful gate before retrying.
19. Generate `AnchorC.PNG` sequentially from `AnchorC.runtime.md` and `TP01_PROMPTS.md` AnchorC prompt rules.
20. Validate `AnchorC.PNG`.
21. If AnchorC fails, run `TP01_FAILURE_SALVAGE.md` failed-but-useful gate before retrying.
22. **STOP POINT 2 — Generated Anchor Report**
    - After generating A / B / C, output a report evaluating:
      - whether AnchorA is layout grammar only
      - whether AnchorB is style DNA only
      - whether AnchorC is asset / object vocabulary only
      - whether the three anchors are sufficiently separated
      - whether any anchor drifted into a finished card, moodboard, design guide, or full story scene
      - whether each anchor follows its runtime and prompt module
      - whether any failed output was reclassified by the salvage gate
    - Do not write `manifest.md`, finalize `SOURCE_INDEX.md`, create a zip, or upload before this report passes.
23. Finalize `SOURCE_DIR/SOURCE_INDEX.md` only after Stop Point 2 passes.
24. Write `manifest.md` only after Stop Point 2 passes.
25. **STOP POINT 3 — Package Preflight Report**
    - Before building the final seed package / zip, output a report confirming:
      - all required files exist
      - source references are recorded
      - runtime files are recorded
      - prompt module version is recorded
      - salvage module version is recorded
      - manifest is aligned with A / B / C and runtime
      - no overview image replaced A / B / C
      - package is complete and ready
26. Generate zip / upload only after Stop Point 3 passes and only if the user requested a package / zip / upload.
27. Report any failed source explicitly.
28. Do not substitute GDrive unless the user explicitly asks for it.

Hard rule:

```text
Do not collapse TP01 into a single uninterrupted generate-and-package run.
Do not generate anchors before AnchorA/B/C runtime exists.
Do not generate anchors without reading TP01_PROMPTS.md.
Do not reuse a failed anchor image as the next base without running TP01_FAILURE_SALVAGE.md.
The stop points and runtime gate are mandatory execution gates.
```

## 6. Anchor Separation Model

### AnchorA.PNG — layout grammar only

Purpose:

```text
Extract page skeleton, reading order, spacing, panels, card hierarchy, and content zones.
For non-card references, extract the source image's spatial grammar: frame ratio, viewpoint, axes, depth layers, node positions, negative space, flow, repetition, and structural rhythm.
```

Must include:

```text
- page ratio or frame ratio
- viewpoint
- primary structure
- depth layers
- dominant axes
- major nodes
- negative space map
- reading / viewing flow
- repetition or rhythm
- structural relationships
```

Must not include:

```text
- complete character
- complete object inventory
- readable final text
- original topic content
- finished illustration
- brand / identity carryover
- style DNA board
- canonical baseline layout copied as the current-run content
```

Failure condition:

```text
If AnchorA looks like a finished infographic, annotated poster, specimen card, reference card, or source-photo explanation board rather than a reusable skeleton, it fails.
If AnchorA reflects the canonical baseline instead of the current source input, it fails.
```

### AnchorB.PNG — style DNA only

Purpose:

```text
Extract visual language without fixing the final layout.
```

Must include:

```text
- material impression
- texture language
- line quality
- edge behavior
- palette family
- contrast profile
- light and shadow mood
- surface character
- overall visual tone
```

Must not include:

```text
- fixed full layout
- complete infographic content
- original story
- object inventory
- workflow logic
- topic conclusion
- moodboard / design guide / brand manual behavior
- material board behavior
- color swatch / palette row behavior
- canonical baseline style copied as the current-run content
```

Failure condition:

```text
If AnchorB becomes a full information card, style guide poster, moodboard, material board, design reference sheet, sectioned grid, or full scene reconstruction, it fails.
If AnchorB reflects the canonical baseline instead of the current source input, it fails.
```

### AnchorC.PNG — asset / object vocabulary only

Purpose:

```text
Extract reusable visual components and object vocabulary.
```

Must include when present in the source:

```text
- poses
- expressions
- gestures
- props
- scene elements
- icons
- arrows
- labels only when essential and minimal
- cards
- reusable UI / infographic components
- primary objects
- secondary objects
- connectors or relationships
- micro details
```

Must not include:

```text
- complete page composition
- complete story
- full hierarchy
- single fixed conclusion
- large unified scene reconstruction
- layout-guide behavior
- style-guide behavior
- brand-specific or identity-specific objects copied from the source
- canonical baseline assets copied as the current-run content
```

Failure condition:

```text
If AnchorC reads like a finished story scene instead of a reusable asset sheet, it fails.
If AnchorC reflects the canonical baseline instead of the current source input, it fails.
```

## 9. Failed-but-Useful Salvage Contract

Before retrying any failed generated anchor, read and apply `TP01_FAILURE_SALVAGE.md`.

Core rule:

```text
A failed image is not automatically trash.
A failed image is also not automatically a valid retry base.
First classify its role.
```

Required gate:

```yaml
failed_but_useful_gate:
  target_role: AnchorA / AnchorB / AnchorC
  verdict_as_target: PASS / PASS_WITH_WARNING / FAIL
  failure_mode:
    - finished_card_drift
    - specimen_card_drift
    - moodboard_drift
    - design_guide_drift
    - material_board_drift
    - scene_drift
    - object_catalog_drift
    - layout_skeleton_drift
    - style_DNA_drift
    - text_or_label_drift
    - source_recreation_drift
  salvageable: yes / no
  salvage_role:
    - negative_diagnostic
    - final_card_draft
    - style_reference_board
    - asset_reference_sheet
    - composition_candidate
    - discard_contaminant
  allowed_next_action:
    - retry_same_role_from_clean_prompt
    - retry_same_role_from_failed_image
    - reclassify_to_downstream_draft
    - store_as_negative_diagnostic
    - discard
```

Reuse rule:

```text
A failed image may be reused as visual base only when it is still inside the same anchor role.
```

Forbidden reuse:

```text
finished card        → not a valid AnchorA/B/C base
specimen card        → not a valid AnchorA/B/C base
moodboard            → not a valid AnchorB base
design guide         → not a valid AnchorB base
material board       → not a valid AnchorB base
full scene           → not a valid AnchorC base
object catalog       → not a valid AnchorA/B base
```

Allowed salvage paths:

```text
negative_diagnostic:
  Use failed output to identify drift pattern and prompt trigger words.
  Do not use failed image as base.

final_card_draft:
  Store separately for downstream final-image generation.
  Do not count it as AnchorA/B/C.

style_reference_board:
  Store separately for human reference only.
  Do not count it as AnchorB.

retry_same_role_from_failed_image:
  Allowed only when target role match is yes, no cross-role drift, no finished-output drift, and repair scope is small.

discard_contaminant:
  Do not use it as base.
  Do not include it in the anchor package except as a diagnostic note when useful.
```

Known anti-regression patterns from 2026-07-02:

```text
AnchorA failure:
  specimen / museum card, title, overview, recognition points, structure map labels, color notes, tags, reference scale.
  Correction: remove TP01 / AnchorA / specimen / card / recognition / overview language; use abstract spatial construction sketch language.

AnchorB failure:
  moodboard / design reference sheet, title, labels, photo tiles, color swatches, tool illustrations, sectioned grid, material board.
  Correction: avoid style DNA / moodboard / reference / board / swatch / palette language; use silent surface atmosphere study language.
```

One-line rule:

```text
Failed anchors may inform the next retry, but they may not become the next anchor base unless they still belong to the same anchor role.
```

## 12. Default Style Policy

Unless the user explicitly gives another style, new TP01 anchor packages should use:

```text
- portrait / vertical layout by default
- soft Japanese hand-drawn card style
- clean readable structure
- low-saturation palette
- fine linework
- white or pale paper background
- rounded modules only when relevant
- social-card readability only when target is a card
```

Default transformation rule:

```text
copy structure, not identity
copy workflow logic, not brand
copy visual function, not exact artwork
copy mood category, not artist-specific style
copy current source extraction through runtime, not canonical baseline content
```

## 13. Execution Order

TP01 must be executed sequentially:

```text
STEP 0  -> read TP01_SOURCE_LOCK.md
STEP 1  -> read Tp01.md / SHA / metadata
STEP 2  -> read TP01_PROMPTS.md / SHA / metadata
STEP 3  -> read TP01_FAILURE_SALVAGE.md / SHA / metadata
STEP 4  -> read raw AnchorA.PNG
STEP 5  -> read raw AnchorB.PNG
STEP 6  -> read raw AnchorC.PNG
STEP 7  -> STOP POINT 1: Anchor Read Report
STEP 8  -> inspect current user reference / task
STEP 9  -> generate AnchorA.runtime.md
STEP 10 -> validate AnchorA.runtime.md
STEP 11 -> generate AnchorB.runtime.md
STEP 12 -> validate AnchorB.runtime.md
STEP 13 -> generate AnchorC.runtime.md
STEP 14 -> validate AnchorC.runtime.md
STEP 15 -> STOP POINT 1R: Runtime Report
STEP 16 -> create SOURCE_DIR / SOURCE_INDEX.md draft if creating a package
STEP 17 -> generate AnchorA.PNG from AnchorA.runtime.md + TP01_PROMPTS.md
STEP 18 -> validate AnchorA.PNG
STEP 19 -> if AnchorA fails, run TP01_FAILURE_SALVAGE.md gate before retry
STEP 20 -> generate AnchorB.PNG from AnchorB.runtime.md + TP01_PROMPTS.md
STEP 21 -> validate AnchorB.PNG
STEP 22 -> if AnchorB fails, run TP01_FAILURE_SALVAGE.md gate before retry
STEP 23 -> generate AnchorC.PNG from AnchorC.runtime.md + TP01_PROMPTS.md
STEP 24 -> validate AnchorC.PNG
STEP 25 -> if AnchorC fails, run TP01_FAILURE_SALVAGE.md gate before retry
STEP 26 -> STOP POINT 2: Generated Anchor Report
STEP 27 -> finalize SOURCE_INDEX.md
STEP 28 -> write manifest.md
STEP 29 -> STOP POINT 3: Package Preflight Report
STEP 30 -> final package validation
STEP 31 -> zip or upload only if explicitly requested
```

Do not skip from reference image directly to final card.

Do not make AnchorB before AnchorA.

Do not make AnchorC before AnchorB.

Do not write manifest before A/B/C and A/B/C runtime exist, unless the user explicitly asks for a planning-only manifest.

Do not zip or upload before Stop Point 3 passes.

Strict gate rule:

```text
Do not skip Stop Point 1.
Do not generate runtime before the Anchor Read Report.
Do not generate anchors before the Runtime Report.
Do not retry failed anchors before the salvage gate.
Do not generate the seed package before the Generated Anchor Report.
Do not zip or upload before the Package Preflight Report.
```

```text
- Do not infer salvage module content from memory.
- Do not infer anchor visual content from memory, prior conversation, or previous seed packages.
```


```text

- Report missing or inconsistent files.
- Do not zip incomplete outputs.
```
