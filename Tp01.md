## 0. Purpose

TP01 is the daily-use anchor extraction workflow for turning one or more reference images into a reusable Anchor Set.

This file is the executable procedure. It must be read from GitHub when the user invokes `TP01` / `tp01` without attaching a local TP01 file.

TP01 no longer uses Google Drive as the default source for the procedure file or seed anchors.

---

## 1. Source Authority

Canonical source order:

```text
1. GitHub repo metadata
2. TP01_SOURCE_LOCK.md
3. Tp01.md
4. raw AnchorA.PNG / AnchorB.PNG / AnchorC.PNG
```


The GitHub connector is used for:


Raw GitHub links are used for:

```text

```

## 2. No-GDrive Default Rule

When the user invokes `TP01` / `tp01` without attachments:

```text
Do not use GDrive as the default TP01 template source.
Do not use GDrive as the default seed anchor source.
Do not search Drive for the TP01 procedure.
Do not search Drive for AnchorA/B/C.
Do not silently fall back to Drive if GitHub raw images fail.
```

GDrive may only be used when the user explicitly says one of the following:

```text
upload to Drive
Drive audit
Drive verification
use GDrive TP01
use Drive anchors
check the Drive folder
```

If the user explicitly requests Drive, that instruction overrides this default.

## 4. Invocation Contract

When the user says `tp01` and does not attach a TP01 file:

1. Fetch repository metadata from `hoiyu915-droid/Tp01`.
2. Read `TP01_SOURCE_LOCK.md`.
3. Read `Tp01.md`, including SHA / metadata.
4. Read AnchorA/B/C from the raw links.
5. Verify all four source components are available:
   - `Tp01.md`
   - `AnchorA.PNG`
   - `AnchorB.PNG`
   - `AnchorC.PNG`
6. Execute the user request using the GitHub-sourced TP01 procedure and anchor set.
7. Report any failed source explicitly.
8. Do not substitute GDrive unless the user explicitly asks for it.

## 5. Output Contract

Each TP01 extraction run produces a complete Anchor Package.

Required outputs:

```text
AnchorA.PNG
AnchorB.PNG
AnchorC.PNG
manifest.md
```

Strict rules:

```text
- AnchorA.PNG, AnchorB.PNG, and AnchorC.PNG must be three separate image files.
- Do not combine A/B/C into one overview image.
- Do not replace the three images with manifest.md.
- Do not crop the original reference directly and call it an anchor.
- Each anchor has one job only.
- Generate complete files directly; do not output patch-only files.
```

Optional local package structure:

```text
TP01_OUTPUT/
  SOURCE_DIR/
    SOURCE_INDEX.md
    REF_01.<ext>
    REF_02.<ext>
  AnchorA.PNG
  AnchorB.PNG
  AnchorC.PNG
  manifest.md
```

If the user asks for a zipped package, produce a complete zip containing the full folder, not partial patches.

## 6. Anchor Separation Model

### AnchorA.PNG — layout grammar only

Purpose:

```text
Extract page skeleton, reading order, spacing, panels, card hierarchy, and content zones.
```

Must include:

```text
- page ratio
- title zone
- main content zone
- visual / character placeholder zone
- supporting note zone
- conclusion / takeaway zone
- arrows, panels, grid, card relationship
- whitespace ratio
- reading order
```

Must not include:

```text
- complete character
- complete object
- readable final text
- original topic content
- finished illustration
- brand / identity carryover
```

Failure condition:

```text
If AnchorA looks like a finished infographic rather than a reusable skeleton, it fails.
```

---

### AnchorB.PNG — style DNA only

Purpose:

```text
Extract visual language without fixing the final layout.
```

Must include:

```text
- line quality
- palette
- typography feel
- icon language
- character proportion
- expression grammar
- border style
- decoration style
- overall visual tone
```

Must not include:

```text
- fixed full layout
- complete infographic content
- original story
- object inventory
- workflow logic
- topic conclusion
```

Failure condition:

```text
If AnchorB becomes a full information card or simply a recolored copy of the reference, it fails.
```

---

### AnchorC.PNG — asset / object vocabulary only

Purpose:

```text
Extract reusable visual components and object vocabulary.
```

Must include:

```text
- poses
- expressions
- gestures
- props
- scene elements
- icons
- arrows
- labels
- cards
- reusable UI / infographic components
```

Must not include:

```text
- complete page composition
- complete story
- full hierarchy
- single fixed conclusion
- brand-specific or identity-specific objects copied from the source
```

Failure condition:

```text
If AnchorC reads like a finished story scene instead of a reusable asset sheet, it fails.
```

## 9. Default Style Policy

Unless the user explicitly gives another style, new TP01 anchor packages should use:

```text
- portrait / vertical layout by default
- soft Japanese hand-drawn card style
- clean readable structure
- low-saturation palette
- fine linework
- white or pale paper background
- rounded modules
- social-card readability
```

Default transformation rule:

```text
copy structure, not identity
copy workflow logic, not brand
copy visual function, not exact artwork
copy mood category, not artist-specific style
```

## 10. Execution Order

TP01 must be executed sequentially:

```text
STEP 0 -> read TP01_SOURCE_LOCK.md
STEP 1 -> read Tp01.md / SHA / metadata
STEP 2 -> read raw AnchorA.PNG
STEP 3 -> read raw AnchorB.PNG
STEP 4 -> read raw AnchorC.PNG
STEP 5 -> inspect user reference / task
STEP 6 -> create SOURCE_DIR / SOURCE_INDEX.md
STEP 7 -> generate AnchorA.PNG
STEP 8 -> validate AnchorA.PNG
STEP 9 -> generate AnchorB.PNG
STEP 10 -> validate AnchorB.PNG
STEP 11 -> generate AnchorC.PNG
STEP 12 -> validate AnchorC.PNG
STEP 13 -> write manifest.md
STEP 14 -> final package validation
STEP 15 -> zip or upload only if explicitly requested
```

Do not skip from reference image directly to final card.

Do not make AnchorB before AnchorA.

Do not make AnchorC before AnchorB.

Do not write manifest before A/B/C exist, unless the user explicitly asks for a planning-only manifest.

## 12. Failure Handling

If any required GitHub source fails:

```text
- Say exactly which file failed.
- Do not silently search GDrive.
- Do not pretend the anchor was read.
- Ask for explicit override only if the user wants to continue with another source.
```

Common failures and corrections:

```text
Tp01.md still mentions Drive as default -> rewrite procedure to GitHub canonical source.
AnchorA too finished -> redo as layout skeleton.
AnchorB includes full layout -> remove fixed composition.
AnchorC forms a complete story -> split into reusable asset vocabulary.
manifest too vague -> add routing, governance, A/B/C separation, validation status.
only one image generated -> split into AnchorA/B/C.
```
